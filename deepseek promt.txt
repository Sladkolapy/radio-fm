# Музыкальное веб-приложение для концентрации, тренировок и медитации

## Общее описание
Создай полнофункциональное веб-приложение для прослушивания музыки с акцентом на концентрацию, тренировки и медитацию. Интерфейс должен быть ярким, современным и интуитивно понятным. Пользователи могут регистрироваться, выбирать музыкальные категории, слушать треки в случайном порядке, искать музыку по тегам, отправлять сообщения разработчику и получать уведомления. Административная панель позволяет управлять контентом, просматривать аналитику и отправлять уведомления пользователям.

## Технологический стек
- **Frontend**: Next.js (App Router), TypeScript, TailwindCSS
- **Backend**: Next.js API Routes
- **Database**: PostgreSQL (Neon Tech)
- **Storage**: Yandex Object Storage
- **Charts**: Chart.js + react-chartjs-2
- **Authentication**: Сессионная авторизация (NextAuth.js или iron-session)
- **Architecture**: Гексагональная архитектура (доменные сущности, порты, адаптеры)
- **Deployment**: Vercel

## Сущности базы данных

### Users
- id (uuid, primary key)
- username (string, unique)
- password_hash (string)
- reset_token (string, nullable)
- created_at (timestamp)
- last_active (timestamp)

### Sessions
- id (uuid, primary key)
- user_id (foreign key)
- token (string, unique)
- expires_at (timestamp)

### Categories
- id (uuid, primary key)
- name (string, unique)
- description (text)
- background_images (array of strings - URLs в Yandex Object Storage)
- created_at (timestamp)

### Subcategories
- id (uuid, primary key)
- category_id (foreign key)
- name (string)
- description (text)
- background_images (array of strings)
- created_at (timestamp)

### Tracks
- id (uuid, primary key)
- title (string)
- category_id (foreign key, nullable)
- subcategory_id (foreign key, nullable)
- file_url (string - URL в Yandex Object Storage)
- duration (integer - seconds)
- created_at (timestamp)
- is_active (boolean)

### Tags
- id (uuid, primary key)
- name (string, unique)

### TrackTags
- track_id (foreign key)
- tag_id (foreign key)

### Messages (от пользователей разработчику)
- id (uuid, primary key)
- user_id (foreign key)
- message (text)
- is_read (boolean)
- created_at (timestamp)

### AdminReplies (ответы админа пользователю)
- id (uuid, primary key)
- message_id (foreign key)
- reply_text (text)
- created_at (timestamp)

### Notifications
- id (uuid, primary key)
- user_id (foreign key, nullable - если null, то для всех)
- title (string)
- content (text)
- is_read (boolean)
- created_at (timestamp)

### Analytics (для отслеживания посещаемости)
- id (uuid, primary key)
- user_id (foreign key, nullable)
- page_visited (string)
- timestamp (timestamp)

## Функциональные требования

### Пользовательская часть

#### Регистрация и авторизация
1. Форма регистрации с полями: имя пользователя, пароль
2. После регистрации генерируется уникальный токен для сброса пароля (показывается пользователю **один раз** с предупреждением сохранить его)
3. Авторизация сессионная (сессия хранится в БД)
4. Форма входа: имя пользователя, пароль
5. Возможность сброса пароля по токену

#### Главный экран
1. Яркий, анимированный интерфейс с фоновыми изображениями, которые меняются в зависимости от категории/подкатегории
2. Отображение всех основных категорий (Концентрация, Тренировки, Медитация и т.д.)
3. Большая кнопка воспроизведения для текущей выбранной категории
4. При нажатии на большую кнопку начинается проигрывание случайного трека из выбранной категории
5. Возможность переключения между категориями и подкатегориями
6. Плеер с элементами управления: play/pause, next, previous, громкость, прогресс-бар
7. Отображение названия текущего трека и категории

#### Воспроизведение
1. Музыка всегда воспроизводится в **случайном порядке** (shuffle)
2. После окончания трека автоматически запускается следующий случайный трек из той же категории/подкатегории
3. При ручном переключении (next/previous) - также случайный трек (не последовательный)
4. Возможность переключения между категориями во время воспроизведения

#### Поиск по тегам
1. Поле поиска с автодополнением из существующих тегов
2. Поиск работает **только по тегам** (не по названиям треков)
3. После выбора тегов находится вся музыка с этими тегами
4. Результаты поиска можно запустить в режиме случайного воспроизведения
5. Поддержка множественного выбора тегов (AND/OR логика)

#### Уведомления
1. Иконка колокольчика в правом верхнем углу
2. При наличии непрочитанных уведомлений - подсветка/точка на колокольчике
3. При клике - выпадающий список последних уведомлений
4. Уведомления могут быть персональными или для всех пользователей
5. Админ может отвечать на сообщения пользователей через уведомления

#### Обратная связь
1. Форма отправки сообщения разработчику (текстовое поле)
2. История отправленных сообщений
3. Получение ответов от админа в виде уведомлений

#### Пожертвования
1. Кнопка "Поддержать проект"
2. Заглушка с сообщением о том, что платежная система будет настроена позже
3. Описание, как можно поддержать разработчиков

### Административная панель (путь /admin)

#### Дашборд с аналитикой (Chart.js)
1. График посещаемости (по дням/неделям/месяцам) - линейный график
2. График популярности категорий (круговая диаграмма)
3. График популярности треков (столбчатая диаграмма)
4. График активности пользователей (по часам)
5. Фильтры по датам

#### Управление музыкой
1. **CRUD операций для треков:**
   - Загрузка аудиофайлов в Yandex Object Storage
   - Указание названия трека
   - Выбор категории и подкатегории
   - Добавление тегов (множественный выбор/создание новых)
   - Редактирование метаданных
   - Удаление трека
2. **CRUD для категорий:**
   - Создание категории с уникальным названием
   - Загрузка фоновых изображений (минимально 2-3 для каждой категории)
   - Редактирование и удаление
3. **CRUD для подкатегорий:**
   - Привязка к родительской категории
   - Свои фоновые изображения
4. **CRUD для тегов:**
   - Создание, редактирование, удаление тегов
   - Просмотр количества треков с тегом

#### Управление сообщениями пользователей
1. Список всех сообщений с сортировкой по дате
2. Флаг прочтения/не прочтения
3. Возможность ответить пользователю (ответ приходит как уведомление)
4. Удаление сообщений

#### Управление уведомлениями
1. Создание нового уведомления:
   - Заголовок и текст
   - Выбор получателей: конкретный пользователь (поиск по имени) или "всем пользователям"
2. Просмотр истории отправленных уведомлений
3. Удаление/редактирование (только неотправленных)

#### Статистика и логи
1. Просмотр активных сессий
2. Логи действий пользователей в реальном времени
3. Метрики использования: сколько треков прослушано, среднее время сессии

## Нефункциональные требования

### Архитектура (Гексагональная)
Должна быть четкая структура папок:
src/
├── domain/ # Доменные сущности и бизнес-правила
│ ├── entities/
│ ├── repositories/ # интерфейсы портов
│ └── use-cases/
├── application/ # Сервисы приложения
├── infrastructure/ # Адаптеры (база данных, storage, API)
│ ├── database/ # PostgreSQL адаптер
│ ├── storage/ # Yandex Object Storage адаптер
│ └── http/ # API клиенты
├── interfaces/ # Контроллеры, API routes, UI компоненты
│ ├── api/
│ ├── components/
│ └── pages/
└── config/ # Конфигурация (.env, constants)


### Безопасность
1. Пароли хешируются (bcrypt)
2. Сессионные токены с истечением срока (7 дней)
3. Токен сброса пароля - одноразовый, показывается только при регистрации
4. Административная панель защищена (специальный флаг у пользователя в БД)

### UI/UX требования
1. Адаптивный дизайн (мобильные устройства, планшеты, десктоп)
2. Анимированные фоны с параллакс-эффектом
3. Плавные переходы между экранами
4. Минималистичный, но яркий дизайн с neon-акцентами
5. Крупные, тактильные элементы управления для плеера
6. Тёмная тема по умолчанию

### Производительность
1. Ленивая загрузка аудиофайлов
2. Предзагрузка следующего трека
3. Кэширование списков треков на клиенте
4. Оптимизация изображений (Next.js Image)
5. Пагинация для длинных списков в админке

### Переменные окружения (.env)

Database (Neon Tech)
DATABASE_URL=postgresql://...

Yandex Object Storage
YANDEX_ACCESS_KEY_ID=your_access_key
YANDEX_SECRET_ACCESS_KEY=your_secret_key
YANDEX_BUCKET_NAME=your_bucket_name
YANDEX_ENDPOINT=https://storage.yandexcloud.net

Authentication
SESSION_SECRET=your_secret_key_here
ADMIN_USER_IDS=uuid1,uuid2 # ID пользователей с правами админа


## Дополнительные требования

### Фоновые изображения
1. Для каждой категории нужно создать как минимум 2-3 фоновых изображения в Yandex Object Storage
2. Изображения в формате WebP, оптимизированы
3. Несколько вариантов для рандомного отображения при загрузке страницы
4. Примеры категорий: "Лесной дождь", "Космическая атмосфера", "Энергичный бас" и т.д.

### Визуальные эффекты
1. Движущиеся частицы на фоне (Canvas или Three.js для легкой анимации)
2. Волновой аудиовизуализатор во время воспроизведения
3. Анимированная кнопка воспроизведения с ripple-эффектом

### Деплой на Vercel
Проект должен быть готов к деплою на Vercel:
1. Настроен git repository
2. Все переменные окружения в Vercel Dashboard
3. Миграции БД запускаются автоматически при деплое
4. Настроены rewrite rules для API routes

## Начальная базовая конфигурация

При первом запуске создать:
1. Администратора (по умолчанию из ADMIN_USER_IDS)
2. Базовые категории:
   - Концентрация (подкатегории: Lo-Fi, Классика, Ambient)
   - Тренировки (подкатегории: EDM, Рок, Хип-хоп)
   - Медитация (подкатегории: Природа, Мантры, Белый шум)
3. Набор тегов: #спокойная #энергичная #инструментальная #быстрая #медленная и т.д.
4. Примеры треков-заглушек для демонстрации

## Ожидаемый результат
Полноценное веб-приложение с понятной структурой, работающей авторизацией, плеером со случайным воспроизведением, админкой с графиками и всем функционалом, описанным выше. Код должен быть чистым, документированным и соответствовать принципам гексагональной архитектуры.
